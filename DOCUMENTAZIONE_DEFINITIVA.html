1. 📖 COS'È QUESTO SISTEMA

Il sistema Pizzeria POS è un'applicazione web che permette di gestire le comunicazioni tra cucina e camerieri in una pizzeria/panineria con 60 tavoli.

Cosa risolve:

  • Il cuoco sa quali tavoli sono in attesa, in lavorazione o pronti
  • Il cameriere vede in tempo reale lo stato di ogni tavolo
  • Storico di tutti i cambi di stato
  • Alert vocale che annuncia quando un ordine è pronto

Cosa NON fa questo sistema:

  • Non gestisce i prodotti/piatti (la pizzeria ha già un sistema per gli ordini)
  • Non gestisce il pagamento
  • Non gestisce la prenotazione dei tavoli
⚠️ Nota importante: Questo sistema è separato dal sistema della pizzeria. La cucina riceve già le comande (cosa hanno ordinato i tavoli) da un altro sistema. Questo serve solo per comunicare lo stato di avanzamento.

3. 🖥️ SCHERMO CUCINA - ISTRUZIONI PER IL CUOCO

3.1 Come accedere

http://IP_DEL_SERVER:8080/cucina.html

Esempio: http://192.168.1.61:8080/cucina.html

3.2 Cosa vedi

  • Griglia 60 tavoli (6 colonne x 10 righe)
  • Statistiche in alto: In attesa / In lavoro / Pronti / Non disp.
  • Ogni tavolo mostra: numero, stato, tempo (se in lavoro)
  • Colori: Grigio=attesa, Arancione=lavorazione, Verde=pronto, Rosso=non disp.

3.3 Come si usa

  1. Selezionare uno o più tavoli: tocca sul tavolo (si evidenzia in blu)
  2. Deselezionare: tocca di nuovo un tavolo selezionato
  3. Azione singola: seleziona solo un tavolo, poi scegli azione
  4. Azione multipla: seleziona più tavoli, poi scegli azione (tutti cambiano insieme)

3.4 Pulsanti azione

PulsanteColoreCosa faVocale
PRONTO🟢 VerdeIl piatto è pronto da portare"Tavolo X, pronto da portare"
IN LAVORAZIONE🟠 ArancioneMostra scelta tempo (10/20/30/45/60 min)"Tavolo X, in lavorazione per Y minuti"
NON DISPONIBILE🔴 RossoProblema momentaneo (es. forno fermo)Nessuna (silenzioso)
ANNULLA⚪ GrigioDeseleziona tutti i tavoliNessuna

4.5 Esempio pratico

Scenario: Tavolo 5 ha ordinato 2 pizze. Cuoco inizia a prepararle.

Azioni:
  1. Tocca sul tavolo 5 ➜ si evidenzia
  2. Clicca "IN LAVORAZIONE" ➜ sceglie "20 min"
  3. Tavolo 5 diventa ARANCIONE con scritto "🔧 In lavoro 20 min"
  4. Il cameriere vede "Tavolo 5 - In lavoro 20 min" sulla CASSA
  5. Dopo 20 min, cuoco tocca tavolo 5 ➜ clicca "PRONTO"
  6. VOCE: "Tavolo 5, pronto da portare"
  7. Cameriere vede "Tavolo 5 - Pronto" e porta l'ordine

3.6 Aggiornamento automatico

La pagina si aggiorna da sola ogni 4 secondi. Non serve ricaricare.

http://IP_DEL_SERVER:8080/cassa.html

4.2 Cosa vedi

  • Griglia 60 tavoli in sola lettura (non puoi cliccare)
  • Ogni tavolo mostra: numero, stato, tempo (se in lavoro)
  • Aggiornamento automatico ogni 5 secondi

4.3 Cosa puoi fare

  • Controllare lo stato di tutti i tavoli a colpo d'occhio
  • Sapere se un tavolo: è in attesa, in lavorazione (e per quanto), pronto, o non disponibile

4.4 Cosa NON puoi fare

  • Non puoi cambiare lo stato (solo il cuoco lo fa)
  • Non puoi selezionare tavoli

9. 🔌 API BACKEND (per sviluppatori)

9.1 Stati tavoli

MetodoEndpointDescrizione
GET/api/stati-tavoliStato di tutti i 60 tavoli
PUT/api/stati-tavoli/:id/con-storicoAggiorna stato e registra storico
GET/api/storico-tuttiTutto lo storico (ultimi 200)

9.2 Esempi pratici

Ottenere stato di tutti i tavoli:

curl http://localhost:3000/api/stati-tavoli

Mettere tavolo 5 in lavorazione per 20 minuti:

curl -X PUT http://localhost:3000/api/stati-tavoli/5/con-storico \
  -H "Content-Type: application/json" \
  -d '{"stato":"lavorazione","tempo_minuti":20}'

Mettere tavolo 5 come pronto:

curl -X PUT http://localhost:3000/api/stati-tavoli/5/con-storico \
  -H "Content-Type: application/json" \
  -d '{"stato":"pronto","tempo_minuti":null}'

11. 🔧 MANUTENZIONE ORDINARIA

11.1 Ogni GIORNO (quando chiudi la pizzeria)

  • Fai il backup del database (comando sopra)
  • Controlla che i container siano attivi: docker ps
  • Verifica che i log non abbiano errori: docker-compose logs --tail=20

11.2 Ogni SETTIMANA (es. lunedì mattina)

  • Riavvia il sistema: docker-compose restart
  • Fai backup completo della cartella
  • Pulisci lo storico se troppo grande: dal browser vai su STORICO → "PULISCI TUTTO LO STORICO"

11.3 Ogni MESE

  • Aggiorna i container: docker-compose pull poi docker-compose up -d --build
  • Verifica spazio su disco: df -h (Linux) o controlla il disco C: (Windows)
  • Elimina backup vecchi (tieni solo ultimi 3 mesi)

13. 🔧 PERSONALIZZAZIONI COMUNI

13.1 Cambiare il testo della voce

Modifica in frontend/cucina.html intorno alla riga 230:

if (stato === 'pronto') parla(`Tavolo ${id}, pronto da portare`);

Cambia con quello che vuoi, es:

if (stato === 'pronto') parla(`Il tavolo ${id} è pronto! Venite a ritirare`);

13.2 Cambiare i colori dei tavoli

Modifica in frontend/cucina.html i CSS:

.stato-pronto { background: #28a745; }  /* Verde */
.stato-lavorazione { background: #fd7e14; }  /* Arancione */
.stato-non-disponibile { background: #dc3545; }  /* Rosso */
.stato-attesa { background: #6c757d; }  /* Grigio */

13.3 Aggiungere un nuovo tempo per "In lavorazione"

Modifica in frontend/cucina.html la sezione dei bottoni:

<button class="btn btn-tempo" onclick="azioneMassivaConTempo('lavorazione', 15)">15 min</button>

13.4 Cambiare IP del server (se cambia rete)

Modifica in TUTTI e 3 i file HTML la riga:

let API_URL = 'http://' + window.location.hostname + ':3000';

Sostituisci con l'IP fisso:

let API_URL = 'http://192.168.1.100:3000';

13.5 Aumentare il numero di tavoli (es. da 60 a 80)

for i in $(seq 61 80); do
  docker exec -i pizzeria_mysql mysql -upizzeria_user -ppizza123 -e \
  "USE pizzeria; INSERT INTO tavoli (id, nome) VALUES ($i, 'Tavolo $i');"
  docker exec -i pizzeria_mysql mysql -upizzeria_user -ppizza123 -e \
  "USE pizzeria; INSERT INTO stati_tavoli (tavolo_id, stato) VALUES ($i, 'in_attesa');"
done
