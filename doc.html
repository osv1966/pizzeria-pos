PIZZERIA POS - DOCUMENTAZIONE TECNICA

ARCHITETTURA

3 container Docker: mysql (porta 3306), backend (porta 3000), frontend (porta 8080)

COMANDI BASE

  • Avvio: docker-compose up -d
  • Fermo: docker-compose down
  • Riavvio: docker-compose down && docker-compose up -d --build
  • Log: docker-compose logs -f

CREDENZIALI

  • MySQL user: pizzeria_user
  • MySQL password: pizza123
  • Database: pizzeria

URL ACCESSO

  • Cucina (touch): http://IP:8080/cucina.html
  • Cassa (monitor): http://IP:8080/cassa.html
  • Storico: http://IP:8080/storico.html

API PRINCIPALI

  • GET /api/stati-tavoli → stato 60 tavoli
  • PUT /api/stati-tavoli/:id/con-storico → aggiorna stato
  • GET /api/storico-tutti → storico cambi

STRUTTURA DATABASE

Tabelle: prodotti, categorie, ordini, dettagli_ordine, tavoli, stati_tavoli, storico_stati

RISOLUZIONE PROBLEMI

  • Connection refused: docker-compose restart
  • Tavoli non si aggiornano: docker restart pizzeria_backend
  • Porta occupata: cambiare porta in docker-compose.yml
